Jarvis Embed SDK

Embed the Jarvis AI Assistant in any web application.

Script Tag (CDN)
<script src="https://jarvis-sandpack.ascendingdc.com/embed-sdk/latest/jarvis-embed.js"></script>
npm
npm install @ascending-inc/jarvis-embed

Quick Start

// With a bundler (ES modules)
import { JarvisEmbed } from '@ascending-inc/jarvis-embed';

const jarvis = new JarvisEmbed({
  provider:    'google',
  token:       googleIdToken,
  containerId: 'chat-container',
  model:       'my-spec',
  onReady:     (jarvisToken) => jarvis.setMcpServers(['my-mcp-server']),
});
// Script tag (no bundler)
const { JarvisEmbed } = window.JarvisSDK;

const jarvis = new JarvisEmbed({
  provider:    'google',
  token:       googleIdToken,
  containerId: 'chat-container',
});

Configuration

OptionTypeDefaultDescription
provider AuthProvider required Auth provider
token string required* OAuth / JWT token. *Not used for hmac.
containerId string — ID of the DOM element to mount the iframe into
container HTMLElement — Direct element reference (alternative to containerId)
width string 100% CSS width of the iframe
height string 600px CSS height of the iframe
apiUrl string https://jarvis.ascendingdc.com Override for self-hosted deployments
model string — Spec identifier sent as ?spec= to the API
debug boolean false Log SDK activity to console
onReady (token) => void — Fires when authenticated. Receives the Jarvis session token.
onError (err) => void — Fires on failure
onMessage (data) => void — Fires on iframe messages to the host page

If neither containerId nor container is provided, the iframe appends to document.body.

Authentication

The SDK calls POST {apiUrl}/api/auth/exchange with your auth payload and receives a Jarvis session token back.

ProviderToken
google Google id_token from OAuth2
s_jwt JWT signed with a shared secret (HS256)
a_jwt JWT signed with a private key (RS256 / ES256)
direct Existing Jarvis session token (skips exchange)
hmac HMAC-SHA256 of userId + timestamp

Methods

MethodDescription
destroy() Removes the iframe and cleans up the window message listener. Call on unmount.
setMcpServers(servers) Activate MCP servers by name. Safe to call before ready — servers are queued and sent on SDK_READY.

MCP (Model Context Protocol)

Pass one or more MCP server names to give Jarvis access to external tools and data sources during a session.

const jarvis = new JarvisEmbed({
  provider:    'google',
  token:       googleIdToken,
  containerId: 'chat-container',
  onReady: async (jarvisToken) => {
    // Discover available servers
    const res = await fetch(`https://jarvis.ascendingdc.com/api/mcp/servers`, {
      headers: { Authorization: `Bearer ${jarvisToken}` },
    });
    const servers = await res.json();
    jarvis.setMcpServers(Object.keys(servers));
  },
});

React

Copy the useJarvis hook from examples/react/src/useJarvis.ts into your project. It wraps JarvisEmbed in a useEffect and calls destroy() on unmount automatically.

import { useJarvis } from './useJarvis';

const jarvisRef = useJarvis({
  provider:    'google',
  token:       googleIdToken,
  containerId: 'chat-container',
  model:       'my-spec',
  onReady:     (jarvisToken) =>
    jarvisRef.current?.setMcpServers(['posthog']),
});

Pass null to defer initialization until the user is authenticated.
